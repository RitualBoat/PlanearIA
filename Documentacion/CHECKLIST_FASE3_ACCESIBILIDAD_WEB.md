# Checklist Fase 3 - Accesibilidad Web Base

Fecha de cierre: 2026-03-29

## Alcance

Validacion de accesibilidad base en modulos raiz y navegacion principal.

## Modulo Home

- [x] Scroll funcional en contenido largo.
- [x] Contraste base legible en tarjetas principales.
- [x] Estructura visual sin recortes en desktop.
- [x] Estados visuales consistentes (cards, acciones rapidas).

## Modulo Planeaciones

- [x] Hub principal con scroll correcto.
- [x] Botones principales con targets adecuados.
- [x] Layout web centrado y con max-width.
- [x] Flujo importar/lista sin recortes reportados.

## Modulo Grupos

- [x] Pantallas principales con jerarquia de cards legible.
- [x] Scroll funcional en vistas largas.
- [x] Contraste de titulos y subtitulos aceptable.

## Modulo Recursos

- [x] Hub y secundarios con estructura responsive consistente.
- [x] Scroll funcional en web/mobile.
- [x] CTA visibles y legibles.

## Modulo Configuracion

- [x] Pildora superior consistente con tabs raiz.
- [x] Acordeones con targets y espaciado adecuados.
- [x] Fade de pildora solo cuando hay overflow real.
- [x] Vista colapsada centrada cuando el contenido cabe en pantalla.

## Navegacion principal

- [x] Tabs persistentes con labels accesibles (`tabBarAccessibilityLabel`).
- [x] Sin doble barra de navegacion en flujos ya migrados.

## Notas

- Este checklist cubre accesibilidad web base (fase 3) y no reemplaza una auditoria WCAG completa.
- Para mejoras avanzadas de accesibilidad, abrir bloque dedicado en roadmap (teclado avanzado, lectores de pantalla por flujo, contraste AA exhaustivo por estado).
